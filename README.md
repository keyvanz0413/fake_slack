# Slackr — 现代敏捷团队在线沟通平台

Slackr 是一款轻量级、响应式的单页 Web 即时通讯应用（SPA）。本项目包含了完整的 Vanilla JS（原生 JavaScript）前端客户端以及基于 Node.js/Express 的高并发轻量级后端服务。

## 📝 项目设计背景与初衷

本项目旨在构建一个高水准的仿 Slack 即时通讯系统，提供轻量、快速的团队协作沟通体验：
*   **原生开发理念**：前端采用纯 Vanilla JS（原生 JavaScript）开发，摒弃重量级框架（如 React/Vue），通过原生 DOM API 与事件驱动机制，实现极致的加载速度与页面响应性能。
*   **服务一体化设计**：后端搭载高可用 Node.js/Express 服务，提供标准的 RESTful API 接口，实现轻量、高内聚的数据处理与存储。
*   **高标准规范**：接口遵循业界主流的 RESTful API 规范，配有完整的 Swagger 交互式文档，前后端规范统一，为模块化扩展打下了坚实基础。

## 🎯 产品需求分析 (PRD)

为了打造一款功能齐备的团队沟通工具，Slackr 的产品功能需求细分为以下核心模块：

### 1. 用户认证模块 (Authentication)
* **用户注册**：新用户可使用邮箱、姓名与密码创建账户。前端需验证邮箱格式、密码长度（至少 8 位），并确保二次确认密码一致。
* **用户登录**：支持邮箱/密码登录，登录成功后在本地安全存储 JWT Token 和基本用户信息。
* **密码找回**：提供忘记密码的交互引导入口。
* **账户登出**：一键清除本地的 Session 状态并自动重定向至登录页。

### 2. 频道管理模块 (Channels)
* **频道分类**：支持 **公开频道 (Public)** 与 **私有频道 (Private)**。公开频道所有人可见且可自由加入；私有频道仅内部成员可见，新成员必须被邀请加入。
* **列表展示**：左侧边栏按公开与私有清晰归类用户加入的频道，提供红点徽标提示未读状态，并支持边栏整体折叠收起。
* **频道创建**：支持快速创建频道，支持设置名称（必填）、描述（选填）和私有化属性。
* **详情与编辑**：允许频道成员查看详细信息（名称、描述、创建时间、创建者）。频道内的用户可随时修改频道的名称与描述，并支持加入/离开频道的操作。
* **成员邀请**：成员可发起邀请。邀请弹窗中将展示尚未加入频道的用户列表（按姓名拼音/字母排序），支持多选复选框一键批量邀请。

### 3. 消息通讯系统 (Messaging)
* **消息展示**：消息区以倒序排列（最新消息展示在底部）。每条消息包含发送者头像、姓名、发送时间，并支持对用户发送的消息显示“已编辑”标签和编辑时间。
* **富文本输入**：支持粗体（`Ctrl+B`）、斜体（`Ctrl+I`）、下划线（`Ctrl+U`）和删除线格式，提供所见即所得的输入框。
* **多媒体消息**：支持上传 Base64 格式的图片附件，图片在聊天流中以缩略图形式展示，点击可打开全屏画廊模态框，并支持左右切换查看频道内的历史图片。
* **消息编辑与删除**：用户可随时编辑或彻底删除自己发送的的历史消息，系统实时无刷更新。
* **表情回应 (Reactions)**：支持对任意消息添加/取消表情回应（👍、❤️、😢、😊、😠 等），并实时合并展示各表情的累计次数与成员名单。
* **消息置顶 (Pin)**：允许将频道内任意重要消息进行置顶，顶部提供全局置顶消息折叠面板，点击可以直接查看置顶消息详情。

### 4. 个人资料模块 (Profiles)
* **资料卡片**：在聊天区点击任何发言用户的名字，可弹窗查看其头像、名字、个人简介（Bio）和邮箱。
* **资料修改**：用户可随时进入个人设置修改姓名、邮箱、简介，并可上传裁剪新的个人头像。
* **密码修改**：允许修改密码，提供显示/隐藏明文密码的切换眼睛图标（默认隐藏）。

### 5. 系统级高级需求 (System & Advanced)
* **实时通知推送 (Toast)**：基于高效的短轮询机制，当用户加入的频道有新消息到来时，右下角弹出精美动画 Toast 提示，显示频道名及新消息数量，5 秒后自动淡出或可手动关闭。
* **离线容灾支持 (Offline Mode)**：系统自动监听网络状态。处于离线状态时，自动切换至离线只读模式（拉取本地缓存的最后一次频道数据），此时所有发信、建群、回应等写操作均被禁用并弹出友好报错提示；网络恢复后自动同步并隐藏提示。
* **基于 Hash 的前端路由 (Fragment Routing)**：利用 `/#channel={channelId}`、`/#profile={userId}` 等 URL Fragment 实现 SPA 的前进、后退和精确页面分享。

## 🛠 技术架构设计

项目采用 **前后端分离** 架构：

```
                              ┌──────────────────────────┐
                              │    浏览器 (Browser SPA)   │
                              └────────────┬─────────────┘
                                           │ HTTPS (JWT)
                                           ▼
┌─────────────────────────────────────────────────────────────────────────────────────────┐
│ 后端服务 (REST API Server)                                                              │
│                                                                                         │
│  ┌───────────────────┐      ┌──────────────────────────┐      ┌──────────────────────┐  │
│  │   Swagger Docs    │      │  Express App (server.js) │      │  JWT Auth Middleware │  │
│  │  (/docs Endpoint)  │      └────────────┬─────────────┘      └──────────┬───────────┘  │
│  └───────────────────┘                   │                               │              │
│                                          ▼                               ▼              │
│                               ┌──────────────────────────────────────────────┐          │
│                               │           Service Layer (service.js)         │          │
│                               └──────────────────────┬───────────────────────┘          │
│                                                      │ JSON Read/Write                  │
│                                                      ▼                                  │
│                               ┌──────────────────────────────────────────────┐          │
│                               │           本地数据存储 (database.json)        │          │
│                               └──────────────────────────────────────────────┘          │
└─────────────────────────────────────────────────────────────────────────────────────────┘
```

### 1. 前端技术栈
- **核心逻辑**：原生 ES6+ JavaScript，无外部重度框架，完全基于原生 DOM API 实现轻量化加载。
- **界面设计**：Vanilla CSS3，引入 CSS 变量控制统一主题色调（雅致马卡龙拼色，主色 `#99CDD8`），使用 Flexbox/Grid 灵活进行自适应移动端适配。
- **缓存与路由**：`LocalStorage` 本地持久化与 Window Hash 事件路由监听。

### 2. 后端技术栈
- **服务引擎**：Node.js & Express 提供高性能 RESTful API。
- **身份校验**：jsonwebtoken (JWT Bearer Token) 拦截受保护路由。
- **数据持久化**：轻量级 JSON 扁平文件数据库，使用 `async-lock` 防止多用户并发写入冲突。
- **API 规范**：集成 Swagger-UI 交互式端点测试平台。

## 🏃 系统安装与部署

### 1. 运行环境要求
- Node.js (v14 或更高版本)
- NPM 或 Yarn 包管理工具

### 2. 启动后端服务器
打开终端，导航至 `backend` 目录，安装依赖包并启动热编译监听服务：
```bash
cd backend
npm install
npm start
```
* 服务器启动后将默认运行在：`http://localhost:5005`。
* 您可以访问 `http://localhost:5005/docs/` 查看交互式 Swagger API 文档并进行接口调试。

### 3. 运行前端应用
前端完全为客户端静态单页应用，您可以使用任何静态服务器运行它。推荐使用 `npx serve`（或 Python 的 `http.server`）：
```bash
# 在项目根目录下运行
npx serve frontend -l 8000
```
启动后，在浏览器中打开 `http://localhost:8000` 即可开始使用。

## 📖 具体使用教程

### 第一步：注册与登录
1. 打开 `http://localhost:8000`，系统默认展示登录页面。
2. **已有测试账户**：
   - 账号：`betty@email.com` / 密码：`cardigan`
   - 账号：`augustine@email.com` / 密码：`august`
   - 账号：`xiaoli@email.com` / 密码：`password123`
3. 输入凭证后点击 **“登录”** 按钮。如果输入格式不正确或密码少于 8 位，页面顶部将弹出红色的中文错误信息框。

### 第二步：主界面交互
1. **左侧边栏**：您会看到已有的频道列表。列表中有一个公开频道 **# 北京美食推荐**。
2. **折叠侧边栏**：点击页面左上角的 **☰** 按钮，可以收起或展开左侧边栏，为右侧聊天窗口腾出最大空间。
3. **未读红点**：当其他用户在非当前活动频道发信时，对应频道名称旁会显示一个精致的红色小圆点，进入频道后小圆点将自动消失。

### 第三步：发送第一条消息与富文本排版
1. 双击左侧的 **# 北京美食推荐** 进入聊天室。
2. 找到聊天窗口底部的富文本输入框，输入文字。
3. 使用鼠标选中部分文字，点击输入框上方的 **B**（加粗）、*I*（斜体）、<u>U</u>（下划线）或 <s>S</s>（删除线）对文本样式进行实时调整，或者使用快捷键（如 `Ctrl+B`）快速排版。
4. 点击右下角的 **“发送”** 按钮（或直接按下回车），您的消息将即时呈现在聊天流的最底部。

### 第四步：发送图片与查看大图
1. 点击输入框下方的 **“图片”** 按钮，选择本地的一张 JPG 或 PNG 图片。
2. 选定后，输入框上方会立即出现图片缩略图预览。
3. 点击 **“发送”**，图片将以缩略图形式发出。
4. 点击聊天信息中的任何图片缩略图，页面将弹出黑暗半透明的 **原图查看器**。您可以使用查看器左右两侧的 `<` 与 `>` 按钮浏览该频道内所有发送过的历史图片，点击右上角 `×` 退出。

### 第五步：表情互动与消息管理
1. **添加表情回应**：鼠标移动到任意一条消息上，点击右侧的 `⋯`（更多操作）按钮，选择 **“表情回应”**，在弹出的 Emoji 选择器中选择 👍 或 ❤️。表情将即时叠加在消息下方，并显示回应的累计人数。再次点击已选择的表情可以取消回应。
2. **置顶重要消息**：在消息的 `⋯` 菜单中选择 **“置顶”**，该消息会立即同步加入窗口顶部的 **“置顶消息”** 折叠栏。点击顶部横幅可以展开查看频道内所有置顶的消息，点击旁边的 `×` 可快速取消置顶。
3. **编辑/删除自己的消息**：对于您自己发送的消息，可以在 `⋯` 菜单中选择 **“编辑”** 重新修改内容，或者选择 **“删除”** 将其撤回。

### 第六步：个人信息设置
1. 点击页面右上角您的用户名（例如：`老张`）。
2. 在弹出的 **“用户资料”** 窗口中，您可以点击 **“上传”** 更换您的头像，或修改姓名和简介（Bio）。
3. 还可以设置新的密码，点击密码框右侧的 **👁️** 图标可以随时在密文和明文状态之间切换。
4. 修改完毕后点击底部的 **“保存修改”**。

### 第七步：离线断网测试
1. 在浏览器开发者工具中，将网络限制（Throttling）调整为 **Offline**（离线状态）。
2. 此时页面顶部会立刻挂起醒目的红色警告横幅：`🔌 离线模式 - 仅能查看缓存内容，无法进行任何操作`。
3. 此时，您依然可以顺畅地上下滚动并查看此前已加载的历史聊天记录（系统已将数据安全缓存至 `LocalStorage`）。
4. 尝试在输入框中发送消息或进行表情点赞，页面会立即阻止操作并抛出报错提示。
5. 将网络重新恢复为 **Online**，横幅将立即消失，轮询服务自动重新上线。

## 沉淀项目 Skills — AI 协作开发实录

本项目作为一个现代化“开源个人项目”，是人机双人协同编程的典型实践。在开发重构过程中，我们通过深度结合并合理调用以下四个核心 AI 技能（Skills），实现了该项目从 0 到 1 的高效搭建：

### 1. BMAD (敏捷架构与目录重组)
*   **敏捷脚手架构建**：BMAD 充当项目架构师角色，定义了轻量前后端一体化的项目骨架，规划了清晰的模块化层级，配置了规范的本地环境及 `.gitignore` 规则，并顺利将远程 Origin 绑定至 GitHub 新库。
*   **结构优化与重整**：优化了前端的静态资源目录及后端的代码入口（`backend/`），确立了规范的微型 API 服务边界，为后续功能开发提供了稳健的项目雏形。

### 2. SuperPower (高阶静态源码扫描与重写)
*   **自动化源码编写**：SuperPower 发挥代码修改的专注力，负责编写了客户端大部分的 Vanilla JS 视图层渲染函数和状态流处理逻辑，并实现了前端全部 67 处中文提示、警告及本地化时间格式化。
*   **代码注释与工程规范**：严格维护代码质量，编写并规范了全量业务逻辑的 JSDoc 注释，提高了纯原生 JS 代码的可读性与工程维护性。

### 3. IA planer (信息架构与页面布局规划)
*   **交互与视口布局构思**：IA planer 负责定义 Slackr 客户端的单页应用（SPA）信息架构。设计了“左侧折叠式频道导航栏 + 右侧主聊天视口 + 顶部置顶消息横幅”的三栏式布局骨架，最大化了有限屏幕空间内的信息呈现效率。
*   **前端路由与状态地图**：设计了基于 URL Hash Fragment 的信息检索路径（如 `/#channel={channelId}` 指向具体频道，`/#profile={userId}` 渲染用户名片），规划了页面状态切换时的视觉过渡路径。
*   **弹出层与信息层级规划**：规划了全局通知 Toast 的弹出轨迹与手动关闭逻辑、创建群组和用户资料模态框（Modal）的弹出时机与表单元素布局，实现了直观易用的用户交互流线。

### 4. Vision explorer (视觉 QA 与人机体验校验)
*   **响应式与自适应布局校验**：Vision explorer 对侧边栏折叠/展开时主聊天框体的响应式 Flexbox 比例、移动端窄屏网格自适应进行了细致校验，保障交互过程中页面无任何视觉错位。
*   **细节交互与图层校准**：校验全屏原图画廊组件左右翻页的对齐美观度，并对离线状态指示器（Offline Banner）的视觉对比度及 Z-Index 堆叠层级进行了反复测试，实现了极佳的视觉表现。
